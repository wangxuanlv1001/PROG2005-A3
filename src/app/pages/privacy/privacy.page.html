Privacy & Security Requirements Mobile Inventory Management System

This page explains how the application protects user input, manages inventory data, and handles updates and deletions responsibly in a mobile environment.

Input Validation

Required fields such as item name, category, quantity, price, supplier name and stock status must be validated before being submitted to the server.

Secure Transmission

The application communicates with the remote server over HTTPS to help protect data during transmission between the device and backend service.

Safe Update & Delete

Update and delete operations should provide clear feedback and confirmation so users understand when they are changing shared inventory data.

Least Privilege

A mobile app should request only the permissions and access necessary for its intended functionality, avoiding unnecessary exposure of sensitive data.

Error Handling

Error messages should be user-friendly and should not expose confidential system or database details that could increase security risks.

Local Data Minimisation

The app should avoid storing unnecessary data locally and should keep only the minimum information needed to support its normal operation.
